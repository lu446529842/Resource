  • Jeff Hardy

    I was getting auth errors trying to create images on EL 6.2, and can confirm a few of the other comments. You must hardcode the macro-looking %WHATEVER% values in the [keystone_authtoken] stanza in glance-api.conf and glance-registry.conf, where sample_data.sh would have set admin_tenant_name = service and admin_user/pass = glance. Additionally, as noted in another comment, the value for --os-auth-url when running glance at the command-line is incorrectly pointing to the glance service itself, and should instead be pointed to keystone on port 35357. Lastly, as is the case elsewhere in the docs, the authentication arguments at the command-line are incorrect: the sample_data.sh value for --os-password = secrete, and --os-tenant-name = demo. If that is all sorted out, you should be able to create images.

    I was getting auth errors trying to create images on EL 6.2, and can confirm a few of the other comments. You must hardcode the macro-looking %WHATEVER% values in the [keystone_authtoken] stanza in glance-api.conf and glance-registry.conf, where sample_data.sh would have set admin_tenant_name = service and admin_user/pass = glance. Additionally, as noted in another comment, the value for --os-auth-url when running glance at the command-line is incorrectly pointing to the glance service itself, and should instead be pointed to keystone on port 35357. Lastly, as is the case elsewhere in the docs, the authentication arguments at the command-line are incorrect: the sample_data.sh value for --os-password = secrete, and --os-tenant-name = demo. If that is all sorted out, you should be able to create images.
